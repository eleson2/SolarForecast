[ ] EV charging detection and schedule integration
    - Determine detection method:
        Option A – Passive: if load > threshold (e.g. > 5 kW) during 21:00–07:00
                   in consumption pipeline, flag as EV session. No extra hardware.
        Option B – Smart charger API (Easee/Zaptec/Wallbox): read scheduled/active
                   sessions and feed into consumption estimates.
        Option C – Manual schedule in config: "EV charges Mon/Wed/Fri 22:00–06:00".
                   Simplest, not adaptive.
    - Key question: does EV + house load ever breach the 10.5 kW nighttime limit?
      If yes, battery must contribute — affects overnight charge strategy.
    - Files: config.js, src/consumption.js, scheduler.js

[~] EV charging detection — smarter outlier removal
    Basic exclusion is implemented: consumption.max_house_w in config filters readings
    above the threshold from the regression (e.g. 5000W excludes EV charging sessions).
    Future enhancement: auto-detect the threshold from the data rather than requiring
    manual config — e.g. flag readings > median + 2×IQR per daytime hour.
    Revisit after 3+ months when enough data exists to estimate a robust per-hour baseline.

[ ] Auto-detect EV charging threshold
    Replace (or cross-validate) the manual consumption.max_house_w with a data-driven
    threshold derived from the accumulated consumption_readings:
    - For each daytime hour, compute median and IQR across all recorded days
    - Flag a reading as EV-contaminated if consumption_w > median + 2×IQR
    - Use the auto-detected threshold to filter getDaytimeConsumptionHistory()
      (fall back to max_house_w config if insufficient data for IQR estimate)
    - Log flagged hours so the user can verify detection is working
    Prerequisite: at least 4 weeks of daytime readings (≥ 28 samples per hour).
